# Frequently asked questions

## Does Shiftframe require a page-builder app?

No. Its core layouts are native Shopify sections, blocks, JSON templates, and theme settings. Apps can still be added through supported app blocks and embeds.

## Can I use my own logo and fonts?

Yes. Upload primary and inverse logos, set the fallback wordmark/monogram, and choose fonts through Shopify font pickers. Custom font files are not bundled with the theme.

## Which preset should I start with?

Choose the preset closest to the catalog and campaign style, not only the favorite color palette. Shiftframe is editorial, Chromalab is energetic, Pantryprint is tactile, and Quietform is restrained.

## Can I change presets after adding content?

Yes, but inspect every template afterward. Preset settings change while store resources, apps, assigned templates, and merchant content remain specific to the store.

## Does Bundle Studio create official Shopify bundles?

It adds chosen variants as separate cart lines. Use Shopify Bundles or a compatible app when component inventory, fixed bundle pricing, subscription bundles, or bundle-specific fulfillment behavior is required.

## Can Drop Room create artificial urgency?

It should not. Configure a real deadline and accurate availability information. Leave countdowns and scarcity messaging disabled when the campaign has no genuine limit.

## Why are related products different between stores?

Related recommendations are generated from each store's product and behavioral data. Complementary products can be configured in Shopify Search & Discovery.

## Does the theme modify checkout?

No. The theme hands the cart to Shopify checkout. Checkout customization depends on Shopify plan and checkout settings.

## Are app integrations guaranteed?

Shiftframe supports Shopify app blocks and standard app embeds. Individual apps can still introduce markup, scripts, styling, or behavior that needs testing or app-vendor support.

## How do I update Shiftframe?

Upload the new release as a separate unpublished theme, review release notes, recreate only intentional code modifications, test the launch matrix, and publish after approval. Never overwrite the only live copy without a backup.

## Is accessibility automatic?

The theme provides accessible implementation patterns, but final accessibility also depends on merchant content, color choices, headings, image alternatives, videos, apps, and custom code.

## How quickly will support respond?

The support target is within two business days for standard requests. Critical reproducible defects are prioritized. See the [support policy](SUPPORT-POLICY.md).
