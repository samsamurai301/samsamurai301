# Shiftframe support policy

## Contact

Use the [Shiftframe support form](https://github.com/samsamurai301/samsamurai301/issues/new?template=shiftframe-support.yml).

- **Theme:** Shiftframe
- **Current documented version:** 1.0.4
- **Support language:** English
- **Response target:** within two business days for standard theme-related requests
- **Critical defects:** reviewed as soon as practical and prioritized for correction

GitHub issues are public. Never post passwords, customer information, API keys, private app credentials, collaborator codes, unpublished commercial information, or security details. Send sensitive security reports privately to [sohangkumar.patel@gmail.com](mailto:sohangkumar.patel@gmail.com?subject=Private%20Shiftframe%20security%20report).

## Included support

- Installation and activation guidance.
- Questions about documented theme settings, templates, sections, and blocks.
- Reproduction and correction of defects in unmodified Shiftframe code.
- Accessibility defect reports related to theme implementation.
- Compatibility investigation against the supported Shopify/browser environment.
- Guidance for updating from an earlier Shiftframe release.

## Outside standard theme support

- Custom design, development, migration, merchandising, photography, copywriting, or data entry.
- Third-party app configuration, defects, or app-specific styling.
- Custom code added by a merchant or another developer.
- Shopify account, billing, checkout, payment gateway, tax, shipping carrier, domain, or platform incidents.
- Performance outcomes materially affected by merchant media, apps, analytics, consent tools, or custom scripts.
- Unsupported browsers or heavily modified/obsolete theme versions.

## Information to include

Provide the `.myshopify.com` store URL, theme version, affected page/preview URL, exact reproduction steps, expected and actual result, browser/device details, screenshots or a recording, recent app/custom-code changes, and whether the issue reproduces in a clean unpublished duplicate.

Never send a store password. When access is required, use Shopify collaborator access with the minimum permissions necessary.

## Severity

- **Critical:** severe security exposure, destructive behavior, or a broadly unusable storefront purchase path.
- **High:** a core shopping path is blocked for a significant customer segment without a reasonable workaround.
- **Normal:** a feature defect with a workaround or limited scope.
- **Low:** cosmetic, content, documentation, or configuration guidance.

## Updates

Theme updates are distributed through the channel authorized by Shopify. Release notes document fixes, behavior changes, and upgrade considerations. Merchants are responsible for reviewing and reapplying intentional custom code when moving to a new theme copy.
